FGA Authorization Model

type inventory_item relations can_view: can_manage from parent can_update: has_clearance and can_manage from parent type inventory_system relations manager: [user] on_vacation: [user] // contextual tuple active_manager: manager but not on_vacation can_manage: active_manager type clearance_level relations next_higher: [clearance_level] granted_to: [user] holder: granted_to or holder from next_higher // hierarchy!

Tuple Types

Stored Tuples

Persistent in FGA store

  • Manager relationship
  • Clearance grants
  • Item hierarchies

Created/deleted based on Okta claims

Contextual Tuples

Passed per-request

  • Vacation status

Not stored - real-time evaluation from Okta claim

Structural Tuples

Pre-seeded (static)

  • Clearance chain (1-10)
  • Item parent relationships
  • Item required clearance
